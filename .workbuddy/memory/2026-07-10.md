# 2026-07-10 - 音频子系统全面修复

## 任务
用户反馈设备：喇叭没有声音，麦克风无法使用，语音申报一直"识别中"。

## 根因（找到 3 个核心 Bug + 1 个次要问题）

### Bug 1: 功放使能引脚电平错误
- 文件：`firmware/ClassPetDevice/ESP32Audio.cpp::init()`
- `digitalWrite(AUDIO_EN_PIN, LOW)` 把 SC8002B/FM8002E 功放关掉了
- 修复：改为 `HIGH`

### Bug 2: 麦克风 I2S 通道未接 ES8311 ADC
- 原 `i2s_set_pin(... data_in_num = I2S_MIC_SD_PIN=GPIO6)`，但 GPIO6 上根本没有麦克风信号
- 实际麦克风数据由 ES8311 ADC 提供，SDOUT 与 DAC SDIN 同一根线
- 修复：让 I2S_NUM_1 与播放端共享 BCLK/LRC，data_in 改为 I2S_DOUT_PIN

### Bug 3: ES8311 ADC/DAC 通路未切换
- 文件：`firmware/ClassPetDevice/ES8311.h`
- 录音时未关闭 DAC、未打开 ADC，寄存器层面一直处于播放态
- 修复：新增 `enableDAC(bool)` / `enableADC(bool)` 静态方法

### 次要: I2S 读取 portMAX_DELAY 阻塞主循环
- 录音时触摸失灵、UI 卡顿
- 修复：改为 50ms 超时

## 设计要点（用于下次参考）
- ES8311 的 SDOUT 与 SDIN 是同一根物理线（BGA 引脚复用），录音和播放不能同时
- 录音前必须 `enableDAC(false) + enableADC(true)`，反之亦然
- SD 卡录音时 `ps_malloc` 优先，无 PSRAM 时降级到 `malloc`
- 上传前 `fileSize <= 44` 提示用户"录音文件为空"以利调试

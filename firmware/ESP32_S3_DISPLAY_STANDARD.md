# 2.8寸 ESP32-S3 Display 开发规范与标准

本文档基于 `lcdwiki` 的 [2.8inch ESP32-S3 Display](https://www.lcdwiki.com/zh/2.8inch_ESP32-S3_Display) 官方资料整理，旨在为本项目的固件开发提供统一的标准。**所有开发者在进行硬件初始化、引脚分配及外设调用时，必须严格遵守本规范**，以防止开发中出现由于引脚冲突或配置错误导致的“莫名其妙”的错误。

---

## 1. 核心硬件参数规范 (N16R8)

开发和配置项目时（如在 `sdkconfig` 中或 Arduino IDE 的 Board 设置中），必须采用以下参数：

- **主控芯片**：ESP32-S3 (Xtensa LX7 32位双核处理器，最高主频 240MHz)
- **Flash 存储**：16MB (外接 QSPI)
- **PSRAM 存储**：8MB (内置 OPI)
- **Wi-Fi/蓝牙**：支持 2.4GHz Wi-Fi (802.11b/g/n) 及 Bluetooth V5.0 (BR/EDR 和 BLE)

### 1.1 强制编译配置宏
由于涉及到复杂的 HTTPS TLS 加解密以及 JSON 解析，默认的 Arduino Loop 任务栈大小会导致栈溢出崩溃。在编译时必须强制加入以下配置：
- **Loop 栈扩容**：`-DARDUINO_LOOP_STACK_SIZE=32768`
*(使用 arduino-cli 编译时需添加参数 `--build-property "compiler.cpp.extra_flags=-DARDUINO_LOOP_STACK_SIZE=32768" --build-path ./build` 以确保宏生效)*

## 2. 硬件外设与引脚分配标准 (CRITICAL)

在进行固件开发时，**严禁随意更改或复用以下已定义好的引脚**。如果擅自更改引脚配置，将导致硬件模块无法正常工作。

### 2.1 液晶屏 (LCD) - 驱动 IC: ILI9341V
- **接口类型**：4-Wire SPI
- **分辨率**：240 x 320
- **颜色深度**：65K (RGB565 常用)
- **引脚分配**：
  - **CS (片选)**：`IO10` (低电平有效)
  - **DC (数据/命令选择)**：`IO46` (高电平: 数据, 低电平: 命令)
  - **SCLK (时钟)**：`IO12`
  - **MOSI (写数据)**：`IO11`
  - **MISO (读数据)**：`IO13`
  - **RST (复位)**：`EN` (与 ESP32-S3 主控共用复位)
  - **BL (背光控制)**：`IO45` (高电平点亮，低电平关闭)

### 2.2 电容触摸屏 (Touch) - 驱动 IC: FT6336G
- **接口类型**：I2C (与扩展 IIC 接口共用)
- **分辨率**：240 x 320
- **引脚分配**：
  - **SDA (数据)**：`IO16`
  - **SCL (时钟)**：`IO15`
  - **RST (复位)**：`IO18` (低电平复位)
  - **INT (中断)**：`IO17` (触摸事件时输入低电平)

### 2.3 音频外设 (I2S) - 包含扬声器与麦克风
- **接口类型**：I2S
- **引脚分配**：
  - **MCLK (主时钟)**：`IO4`
  - **BCLK (位时钟)**：`IO5`
  - **DOUT (输出数据)**：`IO6`
  - **DIN (输入数据)**：`IO8`
  - **LRCLK (左右声道选择)**：`IO7` (高电平: 右声道, 低电平: 左声道)
  - **EN (音频使能)**：`IO1` (低电平使能，高电平禁止)

### 2.4 MicroSD 卡 (TF卡)
- **接口类型**：SDIO (四线制)
- **引脚分配**：
  - **CLK (时钟)**：`IO38`
  - **CMD (命令)**：`IO40`
  - **D0~D3 (数据)**：`IO39` / `IO41` / `IO48` / `IO47`

### 2.5 状态指示灯 (RGB LED)
- **特性**：单线控制，集成红、绿、蓝三种颜色。
- **控制引脚**：`IO42`

### 2.6 系统按键与电池监控
- **BOOT (下载/用户按键)**：`IO0` (上电时拉低进入下载模式，平时可做普通按键)
- **RESET (复位)**：`EN`引脚
- **电池电压检测 (ADC)**：`IO9`

### 2.7 串口通信 (UART0)
- **RXD0**：`IO43`
- **TXD0**：`IO44`

### 2.8 可用扩展引脚 (GPIO)
若需外接其他传感器或设备，**仅允许使用**以下尚未被占用的引脚：
- **普通 IO**：`IO2`, `IO3`, `IO14`, `IO21`
- **扩展 I2C**：`IO16` (SDA), `IO15` (SCL) *(注：与电容触摸屏共用)*

## 3. 开发规范与防坑指南

为了防止出现莫名其妙的错误，开发时需遵守以下原则：

1. **总线共享冲突防止**：
   - `SPI` 总线已专门分配给 LCD。
   - `I2C` 总线 (IO15, IO16) 上挂载了电容触摸屏芯片，扩展连接外部 I2C 设备时，需确认外部设备的 I2C 地址与 `FT6336G` 地址不冲突。
2. **初始化顺序**：
   - 硬件初始化时，应优先初始化 ESP32-S3 系统时钟及电源管理，随后依次初始化 LCD 屏幕驱动、背光、触摸驱动及音频 I2S。
3. **音频外设使能**：
   - 驱动喇叭前，必须确保 `IO1` 处于**低电平**（使能状态），否则音频无输出。
4. **背光控制**：
   - LCD 背光默认由 `IO45` 控制，必须设置为输出模式并输出高电平才能点亮屏幕。推荐使用 PWM (`ledc`) 控制以实现背光亮度调节。
5. **存储与外设初始化**：
   - 启用 PSRAM (8MB OPI) 时，需在配置中开启 OPI 模式，避免系统运行内存不足。
   - 初始化 SD 卡时，需使用 SDIO 模式，而非传统的 SPI 模式。
6. **电源与功耗**：
   - 当屏幕、喇叭同时工作且电池正在充电时，总电流可能高达 `560mA`，在设计通过 USB 供电或外部供电时，电源应能够提供至少 `1A` 的峰值电流以保障稳定性。
